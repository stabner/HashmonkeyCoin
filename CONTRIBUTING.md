# Contributing to HashmonkeyCoin

Thank you for contributing to HashmonkeyCoin! This document provides guidelines for contributors.

## Getting Started

### Development Setup

1. Fork and clone the repository
2. Install dependencies (see README.md)
3. Build and test the project

### Branch Strategy

- **`main`**: Production-ready code (protected)
- **`develop`**: Integration branch for features
- **`feature/*`**: Feature development branches

### Development Workflow

1. Create a feature branch from `develop`
2. Make your changes with tests
3. Commit with conventional commit messages
4. Push and create a Pull Request

## Code Style

- Follow existing code patterns
- Write meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and reasonably sized

## Testing

- All new functions must have unit tests
- Complex features need integration tests
- Run `make check` before submitting

## Pull Request Process

- Target the `develop` branch
- Include tests for new functionality
- Ensure all CI checks pass
- Link related issues in PR description

## Community Guidelines

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn and grow
